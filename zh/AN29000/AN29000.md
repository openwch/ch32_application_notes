# 说明

FMC（Flexible Memory
Controller，可变存储控制器）是微控制器中用于扩展外部存储器的外设模块。FMC支持驱动包括SRAM、SDRAM、NOR
FLASH及NAND
FLASH在内的多种存储器类型，其中SDRAM控制器是FMC的核心功能之一，为需要大容量运行内存的应用场景提供了硬件基础，本应用手册主要介绍了如何使用FMC驱动SDRAM，包括硬件设计和软件配置以及如何同时挂在两个SDRAM使用。本设计仅支持普通SDRAM，不支持DDR类型。

#  基本工作原理

## 信号线说明

SDRAM芯片的外部信号线可分为以下几类，掌握各信号线的功能是正确进行硬件设计和软件配置的基础

| **信号线** | **类型** |                  **说明**                   |
|:----------:|:--------:|:-------------------------------------------:|
|    CLK     |    I     | 同步时钟信号，所有输入信号在CLK上升沿被采集 |
|    CKE     |    I     | 时钟使能信号，禁止时钟时SDRAM启动自刷新操作 |
|    CS#     |    I     |            片选信号，低电平有效             |
|    CAS#    |    I     |   列地址选通，为低电平时地址线表示列地址    |
|    RAS#    |    I     |   行地址选通，为低电平时地址线表示行地址    |
|    WE#     |    I     |            写入使能，低电平有效             |
| DQM\[0:1\] |    I     |            数据输入/输出掩码信号            |
| BA\[0:1\]  |    I     |     Bank地址输入，选择要控制的内部Bank      |
| A\[0:12\]  |    I     |            地址输入（行列复用）             |
| DQ\[0:15\] |   I/O    |              数据输入/输出信号              |

## SDRAM存储结构

SDRAM内部包含多个存储阵列（Bank），每个Bank由行（Row）和列（Column）寻址。以常见的W9825G6KH为例，其内部包含4个Bank，每个Bank具有13位行地址和9位列地址，数据宽度为16位，总容量为4×2¹³×2⁹×16bit
= 32MB。

## SDRAM关键时序参数 

SDRAM的时序参数是驱动配置的核心，以下参数必须与SDRAM数据手册严格对应：

tRCD（RAS-to-CAS Delay）
：行有效命令与读/写列命令之间的延迟周期数，一般为2~3个时钟周期。

tRP（Row Precharge Delay）
：预充电命令到下一个有效命令之间的延迟周期数。

tWR（Write Recovery Time） ：写操作结束到预充电命令之间的延迟周期数。

tRC（Row Cycle Delay）
：连续两个行有效命令之间的最小间隔，以及刷新命令与激活命令之间的间隔。

tRAS（Self Refresh Time） ：自刷新操作的最小SDRAM时钟间隔。

CAS
Latency：发送列地址（READ命令）之后到数据开始输出的SDRAM时钟周期数，通常为2或3，SDRAM控制器配置必须与SDRAM模式寄存器的设置保持一致。

刷新周期：SDRAM需要周期性刷新以维持数据，标准要求每64ms刷新全部行。刷新周期计算公式为：刷新间隔
= 64ms ÷ （行数）

#  硬件设计指南

## FMC与SDRAM引脚连接对照

STM32的FMC外设通过专用引脚与SDRAM芯片连接，下表列出了关键的引脚对应关系：

|   **FMC引脚**    | **SDRAM引脚** |      **说明**      |
|:----------------:|:-------------:|:------------------:|
|  FMC_A\[12:0\]   |   A\[12:0\]   |    行/列地址线     |
|  FMC_A\[15:14\]  |   BA\[1:0\]   |     Bank地址线     |
|  FMC_D\[15:0\]   |  DQ\[15:0\]   | 数据线（16位宽度） |
|  FMC_NBL\[1:0\]  |  DQM\[1:0\]   |    数据掩码信号    |
|    FMC_SDCLK     |      CLK      |    同步时钟信号    |
|    FMC_SDNWE     |      WE#      |      写入使能      |
|     FMC_NRAS     |     RAS#      |     行地址选通     |
|     FMC_NCAS     |     CAS#      |     列地址选通     |
| FMC_SDNE\[1:0\]  |      CS#      |  芯片使能（片选）  |
| FMC_SDCKE\[1:0\] |      CKE      |      时钟使能      |

注释：FMC_SDNE和FMC_SDCKE各有两条信号线，分别对应两个独立的存储区域。使用编号为0的信号线组对应存储器区域1（首地址0xC000
0000），使用编号为1的信号线组对应存储器区域2（首地址0xD000 0000）

## 硬件设计要点

1.  数据总线宽度匹配：如果SDRAM芯片为16位宽度，只需连接FMC_D\[15:0\]前16条数据线，同时在FMC配置寄存器中将数据宽度配置为16位。

2.  时钟信号：FMC_SDCLK的时钟频率可通过FMC_SDCR1寄存器的SDCLK位配置为HCLK的1/2或1/3，与SDRAM通讯的同步时钟最高频率为100MHz。

3.  NBL（字节掩码）信号：对于16位数据总线，需要NBL0和NBL1两个掩码信号；8位总线仅需NBL0；32位总线需要使用全部4个NBL。

4.  PCB布局建议：

    1.  SDRAM应尽量靠近STM32放置，缩短走线长度

    2.  时钟线FMC_SDCLK走线长度应尽量与数据/地址线匹配

    3.  注意信号完整性和电源去耦处理

    4.  避免SDRAM与NAND等设备同时占用FMC总线导致总线冲突

#  软件配置指南

## 配置流程概述

FMC驱动SDRAM的软件配置按以下步骤进行：

1)  引脚时钟配置：配置FMC GPIO 口和时钟源使能

    RCC_HBPeriphClockCmd(RCC_HBPeriph_FMC,ENABLE);

GPIO_Config();

2)  FMC控制器配置：配置SDRAM控制寄存器和时序寄存器，如果需要同时使用两个SDRAM，则需要分开两次配置bank5和bank6,且bank6的配置参数与bank5均为一致，无法单独配置。

FMC_SDRAM_TimingTypeDef SDRAM_Timing={0};

SDRAMInitStructure.FMC_InternalBankNumber=FMC_InternalBankNumber_4;
//SDRAM的bank数

SDRAMInitStructure.FMC_WriteProtection=FMC_WriteProtection_Disable;

SDRAMInitStructure.FMC_SDClockPeriod=1; //时钟分频 SDRAM的CLK=HCLK/
SDClockPeriod

SDRAMInitStructure.FMC_ReadPipeDelay=FMC_ReadPipeDelay_none;

    存储区域，则必须通过将CTB1 和CTB2 位置1

#  常见问题与调试技巧

## 初始化失败排查

当SDRAM初始化失败时，应从以下方面逐层排查：

1.  硬件连接检查：确认FMC接口引脚是否正确配置，检查SDRAM电源电压（通常3.3V）是否稳定

2.  时钟配置验证：确保FMC时钟源来自HCLK，分频系数计算正确

3.  时序参数核对：对照SDRAM数据手册检查TRCD、TWR、TRP、TRC等时序参数是否与手册一致

4.  使用示波器检测：观测FMC_SDCLK信号是否稳定，检查控制信号是否按预期输出

## 读写失败排查

1.  若连续读写SDRAM时出错，但是增加延时后正常，通常是时序配置过快导致的。可尝试降低FMC时钟频率，再逐步调整时序参数相位等

2.  确保正确配置了SDRAM的Bank选择、行列地址位数和数据总线宽度

3.  确认MCU端使用的片选信号与FMC Bank配置对应

4.  检查SDRAM的地址空间是否与数据手册中的容量匹配

## 读写速度实测

通过大批量的读写来测定速度，配置独立的IO口，读/写操作开始拉低IO，读/写操作结束拉高IO，通过计算得出SDRAM的实际读写速度：

32位SDRAM，在SDRAM_CLK=100MHz时，DMA方式256位宽下实际读写速度为380MB/S；与理论上的400MB/S仅存在部分损耗。
